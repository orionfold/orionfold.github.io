> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a slim Proof CTA band to every software and model detail page, pointing back to `/proof/`, with honest copy that differs for products that earned a proof star vs. those that did not.

No test runner in-repo; the one pure-TS unit is TDD'd with a throwaway `node --test` script in the scratchpad (the project's "scratchpad test" precedent), and the integration gate is `npm run build` + in-browser checks on `:4321`.

### Task 1: `isProvenProduct()` helper in `src/data/proof.ts`

**Files:**
- Modify: `src/data/proof.ts` (append exported function near the `cols` export)
- Test: `<scratchpad>/proof-helper.test.mjs` (throwaway; not committed)

**Interfaces:**
- Consumes: the existing `cols: MatrixCol[]` export (each has `ours: boolean` and optional `href`).
- Produces: `export function isProvenProduct(type: 'software' | 'model' | 'book', slug: string): boolean`. Returns `true` iff the page's own path matches the `href` of a `cols` entry with `ours === true`. Path shape: `software` → `/software/<slug>/`, `model` → `/models/<slug>/`, `book` → always `false`.

- [ ] **Step 1: Write the failing test**

Create `<scratchpad>/proof-helper.test.mjs`. Because `proof.ts` has no runtime deps, copy the `cols` array shape into a tiny fixture is NOT needed — instead test against the real logic by importing a compiled copy is overkill. Use a self-contained re-implementation check: the test asserts the *behavior contract* by importing the function via a thin transpile. Simplest reliable path in this repo: write the test to shell out to `astro build`'s type system is too slow. Instead, assert the contract directly with `node --test` against a hand-mirrored copy of the path logic is circular.

Use this concrete approach: the test imports the function from a `.mjs` mirror the implementer pastes the **final** logic into. To keep it honest, the mirror is generated by the test author copying the exact body. Write the test as:

```js
import { test } from 'node:test';
import assert from 'node:assert/strict';

// Mirror of the cols that carry ours:true today (advisor/kepler/patent).
// The test pins BEHAVIOR; Task verifies the real export matches via build.
import { isProvenProduct } from './proof-helper.mjs';

test('proven software (advisor) is proven', () => {
  assert.equal(isProvenProduct('software', 'advisor'), true);
});
test('proven models (kepler, patent-strategist) are proven', () => {
  assert.equal(isProvenProduct('model', 'kepler'), true);
  assert.equal(isProvenProduct('model', 'patent-strategist'), true);
});
test('unproven software (cortex) is not proven', () => {
  assert.equal(isProvenProduct('software', 'cortex'), false);
});
test('unproven model (finance-chat) is not proven', () => {
  assert.equal(isProvenProduct('model', 'finance-chat'), false);
});
test('books are never proven', () => {
  assert.equal(isProvenProduct('book', 'advisor'), false);
});
test('a real model slug that is not an ours-column is not proven', () => {
  assert.equal(isProvenProduct('model', 'ii-medical-8b'), false);
});
```

Then create `<scratchpad>/proof-helper.mjs` with the standalone logic + a fixture `cols` that mirrors the three `ours:true` entries:

```js
const cols = [
  { key: 'advisor', ours: true, href: '/software/advisor/' },
  { key: 'kepler', ours: true, href: '/models/kepler/' },
  { key: 'patent', ours: true, href: '/models/patent-strategist/' },
  { key: 'qwen', ours: false },
  { key: 'llama', ours: false },
  { key: 'deepseek', ours: false },
];

export function isProvenProduct(type, slug) {
  if (type === 'book') return false;
  const base = type === 'software' ? 'software' : 'models';
  const path = `/${base}/${slug}/`;
  return cols.some((c) => c.ours && c.href === path);
}
```

- [ ] **Step 2: Run test to verify it passes (mirror is correct)**

Run: `node --test <scratchpad>/proof-helper.test.mjs`
Expected: all 6 tests PASS (this validates the logic the real export will use).

- [ ] **Step 3: Add the real export to `src/data/proof.ts`**

Append after the `cols` export block (it must come AFTER `cols` is declared). Note `models` base for model type:

```ts
// Proven = the product appears as one of our own (ours:true) columns in the
// matrix above, i.e. it earned a star anyone can rerun. Drives the Proof CTA
// band on its detail page (A4). Books are never part of the proof story.
export function isProvenProduct(type: 'software' | 'model' | 'book', slug: string): boolean {
  if (type === 'book') return false;
  const base = type === 'software' ? 'software' : 'models';
  const path = `/${base}/${slug}/`;
  return cols.some((c) => c.ours && c.href === path);
}
```
